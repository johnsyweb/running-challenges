Version 2.0.1 is out with a proper userscript build, smarter home parkrun detection, and some UI improvements.

## Userscript for Tampermonkey and iOS

If you can’t install the browser extension (for example on iPhone or iPad), you can now use the **userscript** instead. The script is built from the same code as the extension and shares the same version number (2.0.1). Install a userscript manager—e.g. [Tampermonkey]({{ site.data.userscript_managers.tampermonkey_chrome }}) or [Violentmonkey]({{ site.data.userscript_managers.violentmonkey_chrome }}) on desktop, or [Userscripts]({{ site.data.userscript_managers.userscripts_ios }}) or [Stay]({{ site.data.userscript_managers.stay_ios }}) on iOS—then open the script on our website to install it. The [getting started](/getstarted/) page has step-by-step instructions.

## Extension improvements

### Home parkrun and athlete ID from the page

When you haven’t set a home parkrun in options, the extension now infers it from your results. It uses your most recent events so the choice stays relevant. Athlete ID is also inferred from the parkrun page when possible, so the extension can work even when the page URL doesn’t include it in the usual way.

### Challenge map layout

Challenge rows that can show a map no longer reserve a large empty space before you click “show map”. The map area only gets height when the map is actually displayed, so the page is easier to scroll.

### Volunteer Challenges heading

The “Volunteer Challenges” section now has a stable heading ID so it behaves better with assistive tech and our own screenshot tooling.

## Website

- The site now has a favicon.
- The home page and get started flow mention the userscript and iOS-friendly install path.
- Screenshots used on the site have been refreshed to match the current extension UI.
